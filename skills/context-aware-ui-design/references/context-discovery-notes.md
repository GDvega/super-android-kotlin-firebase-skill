# Context Discovery Notes

Use these prompts to make UI design fit the app instead of a generic template.

## Context dimensions

- Domain: what kind of product is this?
- User: who uses it, and under what stress or ability constraints?
- Primary flow: what must happen quickly and safely?
- Data sensitivity: what information should be minimized, masked or confirmed?
- Environment: indoor, outdoor, mobile, one-handed, poor network or shared device?
- Failure modes: what happens when loading, sync, auth, permissions or network fails?

## Design direction

The design recommendation should connect each visual choice to one of those context dimensions.
Avoid changing business logic as part of visual work.
