# UX Audit Rubric

Use this rubric to keep audit findings grounded.

## Severity

- Critical: blocks task completion, hides sensitive risk or causes unsafe action.
- High: causes likely confusion, broken recovery or inaccessible interaction.
- Medium: slows users or increases repeated effort.
- Low: polish or consistency issue with limited task impact.

## Dimensions

- Goal clarity.
- Information hierarchy.
- Primary action visibility.
- State coverage.
- Error recovery.
- Accessibility.
- Adaptive behavior.
- Visual performance risk.

## Evidence

Every finding should reference a file, UI state, missing preview, missing test or observable workflow issue.
